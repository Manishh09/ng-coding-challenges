# v0.2.0 Release Checklist – ngQuest

> **Release theme:** UI & Layout Refresh  
> **Based on:** CHANGELOG.md → v0.2.0

This checklist ensures a safe, repeatable, and production-ready release.

---

## 1. Pre-Release Validation (on `develop`)

### Code & UX

- [ ] New layout renders correctly across pages
- [ ] Sidebar navigation works as expected
- [ ] Empty states display correctly (e.g. Signals category)
- [ ] No broken routes
- [ ] No console errors or warnings
- [ ] Responsive behavior verified (desktop + mobile)

### Content

- [ ] Challenge listing loads correctly
- [ ] No challenge JSON / config errors
- [ ] Existing challenges still visible and readable

---

## 2. CHANGELOG Verification

- [ ] `CHANGELOG.md` contains **v0.2.0**
- [ ] Release notes match the actual UI changes
- [ ] Dates use **ISO format** (`YYYY-MM-DD`)
- [ ] `Unreleased` section does NOT contain v0.2.0 items

Example:

```md
## [v0.2.0] – 2026-06-01
```

---

## 3. Create Release Branch (Recommended)

```bash
git checkout develop
git checkout -b release/v0.2.0
git push origin release/v0.2.0
```

- [ ] Release branch created
- [ ] No new features added after this point

---

## 4. Final Smoke Test (Release Branch)

- [ ] App runs locally or in preview
- [ ] Layout spacing and alignment verified
- [ ] Empty states are intentional and informative
- [ ] Branding and naming consistency verified

---

## 5. Merge to `main` (Production)

```bash
git checkout main
git pull origin main
git merge release/v0.2.0
git push origin main
```

- [ ] Merge completed successfully
- [ ] No conflicts

---

## 6. Vercel Deployment Check (Critical)

- [ ] Deployment triggered from `main`
- [ ] Deployment completed successfully
- [ ] Live site reflects:
  - New layout
  - Empty states
  - Navigation changes

---

## 7. Sync Back to `develop`

```bash
git checkout develop
git merge main
git push origin develop
```

- [ ] `develop` is aligned with `main`

---

## 8. Tag the Release

```bash
git checkout main
git tag -a v0.2.0 -m "ngQuest v0.2.0 – UI & Layout Refresh"
git push origin v0.2.0
```

- [ ] Tag exists on GitHub
- [ ] Tag points to correct commit

---

## 9. Create GitHub Release

- [ ] Go to GitHub → Releases → Draft new release
- [ ] Select tag: `v0.2.0`
- [ ] Target branch: `main`
- [ ] Title: `ngQuest v0.2.0 – UI & Layout Refresh`
- [ ] Paste notes from `CHANGELOG.md`
- [ ] Mark as **Pre-release**
- [ ] Publish release

---

## 10. Post-Release Cleanup

```bash
git branch -d release/v0.2.0
git push origin --delete release/v0.2.0
```

- [ ] Release branch removed

---

## 11. Release Completion Criteria

- [ ] Production reflects the new layout
- [ ] GitHub Release published
- [ ] CHANGELOG matches production
- [ ] `main` and `develop` are in sync

---

✅ **v0.2.0 release is complete when all items above are checked.**
