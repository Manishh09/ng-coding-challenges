💡 How canMatch Works

1

Route Matching Phase

Router evaluates canMatch guards before route matching. If guard returns false, the route is skipped.

2

Lazy Loading Optimization

Premium component bundle is never downloaded for free-tier users. Saves bandwidth!

3

Fallback Route Pattern

Dual routes with same path: first has canMatch guard, second has no guard. Router tries first, falls back to second if blocked.

4

User Experience

Users see upgrade prompt instead of "Access Denied" error. Guard returns UrlTree for redirects.
